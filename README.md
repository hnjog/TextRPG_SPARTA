## Git Branch 작업 팁!

- Main은 최종 결과물로 사용할 예정<br>

- Dev를 집처럼 여기자<br>

- 작업에 따라 Dev를 기준으로 Feature 브랜치를 생성하고<br>
  작업 완료 후, Merge를 통해 Dev에 적용!<br>
  (이후 해당 Feature_Branch를 삭제하면 깔끔!)<br>

- Merge 하기 전에 fetch를 눌러<br>
  받을 데이터 있는지 확인하기!<br>
